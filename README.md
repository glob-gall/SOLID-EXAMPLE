# SOLID

### é conjunto de 5 principios

### SINGLE RESPONSABILITY PRINCIPLE
- cada classe deve ter apenas uma responsabilidade
### OPEN CLOSED PRINCIPLE
- uma classe deve ser aberta para extensões mas não para modificações
### LISKOV SUBSTITUTION PRINCIPLE
- ser capaz de substituir atributos sem quebrar a aplicação
### INTERFACE SEGREGATION PRINCIPLE
- dividir interfaces
### DEPENDENCY INVERSION PRINCIPLE
-  não depender de implementações, usar interfaces
